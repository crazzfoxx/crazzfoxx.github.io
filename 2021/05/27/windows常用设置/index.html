.net框架介绍，IIS服务介绍

.NET相关概念

.NET基础架构

.net standard library定义了一系列标准函数接口

.net framework, .net core, xmarin都实现了该标准定义的接口

asp.net是.net framework框架中所提供，开发web应用程序的类别库，可以运行在安装了.net framework的iis服务器上（Windows Only）或在Mono平台上跨平台运行

.net core是为了微软因应云端化所诱发的多作业平台集成与开发能力特别开发的新一代.net框架。

IIS相关概念

IIS（Internet Information Serives）是一种Web服务组件，其中包括Web服务器、FTP服务器、NNTP服务器和SMTP服务器，分别用于网页浏览、文件传输、新闻服务和邮件发送等方面，它使得在网络（包括互联网和局域网）上发布信息成了一件很容易的事。

2.3 IIS网站配置常见问题

IIS7/iis7.5 HTTP Error 500.19 配置错误由于权限不足而无法读取配置文件

网站物理路径（recursively）需要分别给用户IIS_IUSERS/IUSER/NETWORK SERVICE设置修改权限

在唯一密钥属性”fileExtension”设置为”.mp4”时，无法添加类型为”mimeMap”的重复集合项

为了兼容低版本IIS，档案系统注册了额外的mimeType，一些mimeType在windows10系统是默认注册的，导致重复添加mimeType，无法访问网站的js，css无法加载。

打开网站根目录\web.config，找到如下所示根据情况配置删除。

64位操作系统下提示Microsoft.Jet.OLEDB.4.0未注册

64位操作系统不支持Microsoft OLE DB Provider for Jet驱动程序

第一步，确定系统是否安装了Access

第二步，为IIS网站应用程序启用32位应用程序

若启用32位后，如果访问iis网站报错：

当前标识没有对C:\Windows\Microsoft.NET\Framework\v4.0.30319\Temporary ASP.NET Files的写访问权限

解决：

若C:\Windows\Microsoft.NET\Framework\v4.0.30319下没有Temporary ASP.NET Files文件夹，新建文件夹并给Everyone写权限，注意只给IIS_IUSERS或NETWORK SERVICE写权限仍然报错

COM 类工厂中 CLSID 为 {000209FF-0000-0000-C000-000000000046} 的组件失败，原因是出现以下错误: 80070005 拒绝访问

IIS网站调用office组件失败

配置IIS网站应用程序池：启用32位应用程序并设置标识为LocalSystem
